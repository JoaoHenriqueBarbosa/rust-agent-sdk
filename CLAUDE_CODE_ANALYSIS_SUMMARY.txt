     * Non-concurrent tools (bash, file edits) run alone
     * Bash errors abort siblings
   - Results buffered, emitted in original order

================================================================================
THE HOT PATH (USER PROMPT → RESPONSE)
================================================================================

1. user: query("What files are here?")
   ↓
2. entrypoints/sdk/:query() calls src/query.ts:query()
   ↓
3. query() generator function:
   - getSessionId() [from bootstrap/state.ts]
   - loadTranscriptFile(sessionId) [JSONL read]
   - fetchSystemPromptParts(model)
   - createUserMessage(prompt)
   - normalizeMessagesForAPI(messages)
   ↓
4. callClaude({client, messages, system, tools, ...})
   - Create client via services/api/client.ts
   - Stream via client.beta.messages.stream(params)
   ↓
5. Process streaming events:
   - content_block_delta → yield StreamEvent (text)
   - tool_use → StreamingToolExecutor.addTool()
   ↓
6. Stream ends:
   - stream.finalMessage() → BetaMessage
   - normalizeContentFromAPI() → AssistantMessage
   ↓
7. Persist + continue:
   - appendEntry(AssistantMessage)
   - For each tool: appendEntry(ToolResult)
   - flushSessionStorage() → JSONL write
   ↓
8. On tool_use:
   - StreamingToolExecutor.getRemainingResults()
   - Yields progress + final tool results
   - Next iteration sends tool_results back to API
   ↓
9. Repeat until stop_reason = "end_turn"
   ↓
10. yield AssistantMessage (final)

================================================================================
CRITICAL TYPE TRANSFORMATIONS
================================================================================

Input: "string"
  ↓ createUserMessage()
Internal: UserMessage {uuid, content[], parentUuid, cwd, timestamp}
  ↓ normalizeMessagesForAPI()
API: MessageParam[] {role: "user", content: ContentBlockParam[]}
  ↓ client.beta.messages.stream(params)
Stream: BetaRawMessageStreamEvent
  ↓ stream.finalMessage()
API: BetaMessage {content: BetaContentBlock[], stop_reason, usage}
  ↓ normalizeContentFromAPI()
Internal: AssistantMessage {uuid, message: BetaMessage, created_at}
  ↓ appendEntry()
Persistent: TranscriptMessage (JSONL line in ~/.claude/projects/...)

================================================================================
KNOWN COMPLICATIONS
================================================================================

1. STREAMING TOOL EXECUTION
   - Not sequential! Tools start as they arrive mid-stream
   - Concurrency control needed
   - Results buffered, emitted in receipt order

2. MESSAGE TYPE SCATTERING
   - No single message.ts file!
   - Types defined across:
     * src/utils/messages.ts (main definitions)
     * src/types/logs.ts (persistence types)
     * src/entrypoints/sdk/coreTypes.ts (public API types)
   - Imports reference each other circularly

3. STREAMING FALLBACK
   - If HTTP streaming fails mid-stream, switch to polling
   - StreamingToolExecutor.discard() called
   - Results abandoned, retry from before
   - Tracked in src/query.ts:streamingFallbackOccured

4. COMPACT BOUNDARIES
   - Long sessions trigger message compression
   - Affects resume + token counting
   - Complex logic in src/services/compact/

5. THINKING MODE
   - Extended thinking creates synthetic blocks
   - Need special handling for display
   - Reduces output tokens available

6. PERMISSION SYSTEM
   - Tools blocked until user approves
   - Affects retry flow and early exit
   - Tracked in src/utils/permissions/
